## Compact rules (inyectables a sub-agents)

Las siguientes reglas se cachean para inyectar en cada sub-agent que toque codigo Angular/TypeScript en este proyecto. **No reemplazan leer las skills completas** — son resumen operativo.

### Angular (de `angular-core` + `angular-architecture` + `angular-forms`)

- **Standalone components** siempre. Sin NgModules.
- **Signals** para estado reactivo. `signal()`, `computed()`, `effect()`. RxJS solo cuando hace falta.
- **Control flow nuevo**: `@if`, `@for`, `@switch`. No `*ngIf` ni `*ngFor`.
- **Inject** function en vez de constructor injection cuando aplique.
- **Scope Rule**: componente usado por 1 feature -> vive en esa feature. Usado por 2+ -> sube a `core/shared/`.
- **File naming**: kebab-case. `set-logger.component.ts`.
- **Container vs Presentational**: pages orquestan (inyectan use cases), components renderizan.
- **Forms**: Signal Forms si disponible y conviene, sino Reactive Forms.

### TypeScript (de `typescript`)

- **Strict mode** siempre.
- **Discriminated unions** > clases con campos opcionales para polimorfismo.
- **Value Objects** para invariantes de dominio (Reps, Weight).
- **No `any`**. `unknown` + narrowing si hace falta.
- **Type narrowing** via `switch` exhaustivo.

### Workflow (de `github-pr` + Luca's CLAUDE.md)

- **Conventional commits**: `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`, `test:`.
- **Commits en español** (precedente Politica Gremial + Sendo).
- **NUNCA** "Co-Authored-By: Claude" en commits.
- **NUNCA** build despues de cambios (decision Luca).
- **Calidad > velocidad** SIEMPRE. Conceptos > codigo.

### Arquitectura Forge (de vault `Projects/Forge/v2/arquitectura.md`)

- **Screaming Architecture** por afuera: `features/{onboarding,routines,exercises,training,progress,profile}/`.
- **Clean / Hexagonal por feature**: `{domain,data,ui}/` con dependencias hacia el dominio.
- **`domain/` no importa**: Angular, RxJS, Dexie, nada externo. TypeScript puro.
- **`data/` implementa interfaces de `domain/`**. Adaptadores Dexie viven aqui.
- **`ui/` consume use cases via DI**. No importa `data/` directo.
- **Polimorfismo de WorkedSet/TargetSet**: discriminated union por `TrackingType`.

### TDD (de Luca's CLAUDE.md `Strict TDD Mode: enabled`)

- **Test antes que codigo** en domain layer (use cases, value objects, domain services).
- **UI tests** vienen despues — primero un dominio solido.
- **Cobertura prioritaria**: PR detection, 1RM calculator, validacion de Value Objects, use cases.
